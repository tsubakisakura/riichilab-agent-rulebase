# 本リポジトリの目的

[RiichiLab](https://riichi.dev/) と接続し、生の入出力ログを得ることが本リポジトリの目的です。
そのために、シンプルなwebsocketの接続と、単純なルールベースAIにより、妥当なログを取得します。
強いAIである必要はありませんが、極端に変な打ち方では他の人の迷惑にもなりますので、基本的なルールベースAIを実装します。

# 本リポジトリで行わないこと

* ログのリプレイによるテストなどは行いません。本リポジトリの責務外とします。
* 3人麻雀向けには対応しません。

# リポジトリ構成

* riichilab-agent-rulebase-cli (CLIで起動するagent)
* riichilab-agent-protocol (DTO)

# セットアップ

トークンはCLI引数ではなく、Git管理外の `.env` で渡す想定です。

~~~
cp .env.example .env
~~~

`.env` に発行したトークンを書き込みます。

~~~
RIICHILAB_BOT_TOKEN=replace-with-your-bot-token
~~~

CLIは起動時に自動で `.env` を読み込みます。

# コマンド

~~~
# validation URLへ接続
cargo run validate

# ランク戦へ接続
cargo run ranked

# ランク戦を10試合連続で回す
cargo run ranked -- --games 10
~~~

必要ならログ出力先を上書きできます。

~~~
cargo run validate -- --log-dir out/logs
~~~

# AIのルール

下記ルールを上から順にテストしていき、条件に合致するものを選択します。

* あがれるならあがります。
* 立直できるなら立直します。
* 他家にリーチが掛かっていて、２シャンテン以上であり、立直者の現物を持っている場合、必ず現物を切ります。
* 暗槓/加槓してシャンテンが減らないならば暗槓/加槓します。
* 役牌を鳴ける状況で、かつシャンテンが進むならば、ポンします。
* 役牌を鳴いているか役牌暗刻を確保していて、かつシャンテンが進むならば、チー/ポンします。
* 役牌を鳴いているか役牌暗刻を確保していて、かつシャンテンが減らないならば、大明槓します。
* 上記のどれにも該当しない場合、シャンテン数と受け入れ数を見て、最も良い(シャンテン,受け入れ)の牌を切ります。差が無ければランダムで選択します。

# ログの出力フォーマット

jsonlフォーマットになります。

~~~
{"dir":"in", "data": <<<実際の入力フレーム>>>}
{"dir":"out", "data": <<<実際の出力フレーム>>>}
~~~

ログは半荘ごとに記録されます。つまり、start_gameからend_gameまでを1ファイルとして扱います。
デフォルトの出力先は `logs/` です。
